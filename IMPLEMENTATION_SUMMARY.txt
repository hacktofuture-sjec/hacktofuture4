**File:** [backend/routers/incidents.py](backend/routers/incidents.py) (lines 197-201)

Now properly prefers stored diagnosis:
- Checks `incident.diagnosis` first (from /diagnose endpoint)
- Falls back to request body diagnosis
- Only generates fresh as last resort
- Prevents redundant diagnosis runs

## 🔄 Pipeline Flow

```
Monitor Stage (collects snapshot)
    ↓
POST /incidents/{id}/diagnose  ← NEW ENDPOINT
    ↓
Diagnose Stage (analyzes with fingerprints)
    ↓
POST /incidents/{id}/plan  ← UPDATED TO PREFER STORED DIAGNOSIS
    ↓
Plan Stage (generates actions)
```
